# Technical Change History

- 2025-09-02: Initialized fresh rgx workspace (Cargo workspace, rgx-core, rgx-cli). Added scaffolding modules and CLI. Set repository to https://github.com/rdje/rgx.
- 2025-09-02: Finalized project vision - Lua + JavaScript code execution in regex patterns, Apache 2.0 license, beat PCRE2 performance goal.
- 2025-09-02: Implemented comprehensive lexer and AST system. Created complete token set for all Perl regex features including code blocks (?{lua:...}), (?{js:...}). All lexer tests passing.
- 2025-09-03: Built recursive descent parser and basic VM execution engine. Implemented complete AST-to-bytecode compilation pipeline.
- 2025-09-04: **MAJOR**: Created zero-cost parser abstraction for seamless PGEN integration. Fixed critical VM quantifier bug - \d+ patterns now compile correctly (DigitAscii vs Any). Test improvements: 29 passed, 2 failed (was 3 failed). Parser selection via compile-time feature flags with zero runtime overhead.
- 2025-09-07: **MILESTONE**: Completed high-performance regex virtual machine with comprehensive bytecode execution engine. Implemented 132-opcode instruction set with SIMD optimization support, backtracking with memoization, and alternative tracking. Built optimizing compiler with multi-pass compilation (analysis, optimization, codegen). All 12 VM tests passing covering characters, character classes, quantifiers, alternation, capture groups, and anchors. Fixed alternation tracking and jump offset calculations. VM successfully handles complex patterns like cat|dog, (a*)(b+), \d+, anchors, word boundaries. Performance-optimized bytecode layout with runtime SIMD capability detection. Ready for high-level API integration and PCRE2 benchmarking.

- 2025-10-06: **PERFORMANCE BASELINE ESTABLISHED**: Completed comprehensive PCRE2 vs rgx benchmarking across 7 pattern types and 4 input sizes (100B to 100KB). Established performance baseline: PCRE2 100-450x faster (expected - rgx is new vs PCRE2's decades of optimization). Added pcre2 crate as dev dependency for rgx-bench only (no core dependency). rgx compilation 3.7x slower but throughput shows SIMD potential for complex patterns. Benchmark infrastructure ready for future optimization validation.

- 2025-10-06: **LUA INTEGRATION FOUNDATION**: Implemented complete Lua code execution infrastructure for (?{lua:...}) patterns. Built sandboxed LuaEngine with security features (no I/O, no system access, no module loading, memory limits). Created ExecutionManager for multi-language coordination. Added ExecContext for match data access (captures, position, text, named groups). Thread-safe design with Arc<Mutex<Lua>> wrapper. Core architecture complete - compilation issues with mlua thread safety remain to be resolved. (?{lua:...}) patterns ready for integration once compilation fixed.
