# rgx-capi — C ABI Stability Contract

**Status: pre-1.0 (current workspace version `0.1.0`).** Before `1.0.0`
the surface is still being shaped (Phases 2–7 of
`docs/A9_LANGUAGE_BINDINGS_DESIGN.md` add captures, iterators,
replace, safety limits, embedded scripting, observers, `tail_file`).
This document defines the contract that governs how that surface may
change — both now and after `1.0.0` — so external, non-Rust consumers
who load `librgx.{so,dylib,dll}` (and *cannot* recompile against a
changed signature) know exactly what they can rely on.

This is the contract the Publish-Readiness criterion #1 ("API
contract stabilization") requires for the highest-risk surface. The
Rust-side (`rgx-core`) and CLI contracts are tracked separately and
are out of scope here.

---

## 1. What is covered

The **stable surface** of `rgx-capi` is exactly:

1. The set of exported `extern "C"` **function symbols** and their
   **signatures** (parameter count, order, C types, return type).
2. The **error-code constants** (`RGX_OK`, `RGX_ERR_*`) — their names
   *and* their integer values.
3. The **layout of any `#[repr(C)]` struct** that crosses the
   boundary by value or whose fields the header exposes.
4. The **observable behavioural contract** of each function as
   documented in its header doc-comment (return-code meaning,
   out-param population, null-pointer handling, panic safety,
   threading, memory ownership).
5. The **`MAJOR.MINOR.PATCH`** triple reported by
   `rgx_runtime_version_{major,minor,patch}` and the matching
   `RGX_VERSION_*` header constants.

The generated header `rgx-capi/include/rgx.h` is the authoritative
serialization of 1–5 and is committed to the tree (regenerated by
`cargo build -p rgx-capi` via `build.rs` + `cbindgen.toml`, with
`sort_by = "Name"` so diffs are stable and reviewable).

## 2. What is explicitly NOT covered (free to change any time)

- **Opaque pointer internals.** `RgxRegex` is a forward-declared
  `typedef struct RgxRegex RgxRegex;`. Its size, fields, and the
  Rust type behind it (`Arc<rgx_core::Regex>` today) are *not* part
  of the contract. Never `sizeof`, allocate, copy, or inspect it;
  only hold and pass the pointer.
- **The engine implementation.** Which tier matches a pattern
  (DFA / Pike-VM / TDFA / JIT / interpreter), and the resulting
  performance characteristics, are not contractual.
- **rgx-core's internal API, opcodes, AST, and the PGEN grammar.**
  None of these are exposed across the C ABI; they evolve freely.
- **`rgx_last_error()` message text.** A diagnostic string for
  humans/logs. Its *presence* (non-empty after a failing call,
  empty after success) is contractual; its exact wording is not —
  PGEN's error formatting is its own surface.
- **Unused reserved error-code values.** The `-4` and `-6` gaps in
  the enumeration are intentionally reserved; consumers must treat
  any negative return as "error, consult `rgx_last_error()`", not
  switch exhaustively on today's known set.

## 3. SemVer mapping (the change → version rule)

Versioning is SemVer on the shared library / crate version (the
workspace `version`, surfaced at runtime via
`rgx_runtime_version_*`):

| Change | Bump |
|---|---|
| Remove or rename an exported function | **MAJOR** |
| Change a function's signature (param count/order/type, return type) | **MAJOR** |
| Change the integer value of an existing `RGX_*` constant | **MAJOR** |
| Reuse a previously-assigned-then-removed error-code value for a new meaning | **MAJOR** |
| Change a `#[repr(C)]` struct's layout (field add/remove/reorder/retype) | **MAJOR** |
| Tighten/alter a documented behavioural contract in a way that can break a conforming caller (e.g. a return code now means something else) | **MAJOR** |
| **Add** a new exported function | **MINOR** |
| **Add** a new error-code constant at a fresh unused value | **MINOR** |
| Add a new `#[repr(C)]` struct used only by new functions | **MINOR** |
| Loosen a contract in a backward-compatible way (accept input previously rejected) | **MINOR** |
| Bug fix that makes behaviour match its documented contract | **PATCH** |
| Internal/engine/perf change with no surface effect | **PATCH** |
| Doc-comment clarification with no behavioural change | **PATCH** |

Pre-1.0 caveat: while the version is `0.x.y`, a MAJOR-class change
increments the **MINOR** (`0.x` → `0.(x+1)`) and a MINOR/PATCH-class
change increments the **PATCH**, per Cargo's `0.x` SemVer semantics.
The *classification* above still governs; only the digit that moves
differs until `1.0.0`. Consumers pinning a pre-1.0 `.so` must pin the
exact `0.x` and re-validate on any `0.x` bump.

## 4. The append-only / never-renumber rule (error codes)

The error-code enumeration is **append-only and never renumbered**:

- An assigned value (e.g. `RGX_ERR_INVALID_PATTERN = -2`) keeps its
  meaning **forever**. It is never reused for a different meaning,
  never renumbered, even across MAJOR versions.
- A retired error condition keeps its reserved value (the constant
  may be documented as deprecated; the number is burned, not
  recycled).
- New conditions take the next fresh unused negative value. The
  current gaps (`-4`, `-6`) are reserved space for future
  categories (e.g. text-buffer / required-no-match distinctions);
  declaring them is deliberate so the enumeration stays append-only.
- `RGX_OK` is `0` and immovable. All errors are negative. Callers
  MUST test `rc != RGX_OK` (or `rc < 0`), not enumerate.

Current assignments (Phase 1):

| Constant | Value | Stability |
|---|---|---|
| `RGX_OK` | `0` | stable |
| `RGX_ERR_NULL_POINTER` | `-1` | stable |
| `RGX_ERR_INVALID_PATTERN` | `-2` | stable |
| `RGX_ERR_INVALID_UTF8` | `-3` | stable (reserved-active; surfaced broadly once a bytes API lands) |
| `RGX_ERR_LIMIT_EXCEEDED` | `-5` | stable (reserved-active; returned once Phase 3 safety limits land) |
| `RGX_ERR_INVALID_HANDLE` | `-7` | stable |
| `RGX_ERR_INTERNAL` | `-99` | stable (a caught Rust panic — indicates a bug; please report) |

`-99` is pinned as the sentinel for "unexpected internal failure";
new ordinary categories fill the low negative range, never `-99`.

(Note: rgx-core's *VM opcodes* are sometimes also described as
"append-only / never-renumber". That is a separate internal
invariant — opcodes are **not** part of this C ABI and are not
governed by this document.)

## 5. Per-function stability tiers

Every exported function is tagged **stable**, **experimental**, or
**internal**. The tier is recorded here (the header doc-comment is
the behavioural contract; this table is the stability promise):

| Function | Tier | Notes |
|---|---|---|
| `rgx_compile` | stable | Phase 1. Signature frozen. |
| `rgx_regex_free` | stable | Phase 1. `free(NULL)`-safe; double-free is UB (mirrors C). |
| `rgx_regex_retain` | stable | Phase 1. Cheap `Arc` clone; each handle freed independently. |
| `rgx_is_match` | stable | Phase 1. |
| `rgx_find_first` | stable | Phase 1. Span as `(start,end)` byte offsets. |
| `rgx_last_error` | stable | Phase 1. Borrowed thread-local; valid until the next `rgx_*` call on the same thread; never freed by the caller. |
| `rgx_runtime_version_major` / `_minor` / `_patch` | stable | Phase 1. Pair with `RGX_VERSION_*` header constants to detect header/library skew at load time. |

- **stable**: covered by §3/§4. Breaking it is a MAJOR-class change.
- **experimental**: shipped for feedback; may change with only a
  MINOR bump and a `CHANGES.md` note. Each such function's header
  doc-comment must say "Experimental". (None in Phase 1.)
- **internal**: not for external use even though the symbol may be
  visible (e.g. the `rgx_runtime_*` JIT trampolines `rgx-core`'s c1
  module exports for `dlsym`). These are not part of this contract;
  do not call them.

New phases (2–7) land their functions as **stable** unless their
doc-comment explicitly says "Experimental".

## 6. Deprecation policy (concrete timelines)

1. To retire a stable function or constant, first mark it
   **deprecated** in its header doc-comment (`@deprecated since
   X.Y.Z: use <replacement>`) and add a `CHANGES.md` entry. It keeps
   working unchanged.
2. A deprecated symbol survives **at least 2 subsequent MINOR
   releases** (post-1.0) or **2 subsequent `0.x` releases** (pre-1.0)
   before it may be removed, and removal only in a MAJOR-class
   release.
3. Error-code constants are **never removed** (see §4) — only
   documented as deprecated; the value stays burned forever.
4. The replacement must exist and be stable **before** the
   deprecation is announced, so no window exists with no supported
   path.

## 7. The header-drift CI gate (contract)

`rgx-capi/include/rgx.h` is committed and machine-generated.
**Any change to its meaningful content (declarations, signatures,
constants, struct layouts) without a corresponding workspace
`version` bump is a contract violation and MUST fail CI.**

Gate contract:

- On every PR/push, regenerate the header (`cargo build -p
  rgx-capi`) and assert the committed `rgx-capi/include/rgx.h` is
  byte-identical to the regenerated one (catches "forgot to commit
  the regenerated header"); **and**
- if the committed header's meaningful content differs from the
  header at the merge-base, assert the workspace `version` also
  differs (catches "changed the ABI without bumping the version").
- The cbindgen `header = """…"""` banner is pinned to the project +
  the ABI-stability note; bump it in lockstep with a MAJOR-class
  change.

**Implemented.** `scripts/check-capi-abi.sh` enforces both checks
and is wired into `scripts/run-local-ci.sh` — which *is* the CI
gate (the "Workspace checks" job runs `run-local-ci.sh`), so the
gate runs on every PR/push and is receipt-guarded by the
pre-commit hook. "ABI-meaningful content" is the header minus the
pinned banner and the cbindgen tool-version stamp; the check is
deliberately conservative (any other header change, doc comments
included, requires a workspace `version` bump — the header is
machine-generated, so every content change is a deliberate event).
When neither `origin/main` nor `main` is reachable (initial /
detached checkout) the version comparison is skipped with a clear
message; the byte-identical regeneration check always runs.

## 8. Memory & threading invariants (contractual)

These are part of the behavioural contract (§1.4) and breaking them
is MAJOR-class:

- All Rust-owned objects cross as **opaque pointers**; the caller
  owns the lifetime and calls the matching `rgx_*_free` exactly
  once. `free(NULL)` is a no-op; double-free is UB.
- `rgx_regex_retain` yields an independent handle; each is freed
  separately (the underlying engine is `Arc`-shared).
- Byte buffers (patterns, text, …) are **borrowed for the duration
  of the call**; rgx-capi never takes ownership and never retains
  them past return. Every byte API takes an explicit
  `(const uint8_t*, size_t)` pair — **no NUL-terminated C strings**
  for data (only `rgx_last_error`'s returned diagnostic is a
  NUL-terminated, read-only, borrowed `const char*`).
- `RgxRegex*` is **safe to use from multiple threads for reading**
  (`rgx_is_match` / `rgx_find_first`); `rgx_core::Regex` is
  `Send + Sync`.
- `rgx_last_error` is **per-thread**; a thread only ever sees its
  own most-recent message.
- **No Rust panic crosses the boundary.** Every entry point wraps
  its body in `catch_unwind`; a panic becomes `RGX_ERR_INTERNAL`
  with the message in `rgx_last_error()`. This is contractual: a
  conforming caller may rely on never receiving a foreign unwind.

## 9. How to evolve the surface (checklist for contributors)

When changing `rgx-capi`:

1. Classify the change against §3. Decide the version digit that
   must move.
2. If it touches the error enumeration, obey §4 (append-only,
   never renumber, `-99` reserved).
3. Tag any new function's tier (§5) in its header doc-comment.
4. If retiring anything, follow §6 (deprecate first, ≥2 releases,
   replacement ships first).
5. `cargo build -p rgx-capi` to regenerate `include/rgx.h`; commit
   the regenerated header in the same commit.
6. Bump the workspace `version` per step 1; add a `CHANGES.md`
   entry and update this document if a tier/policy changed.
7. The header-drift gate (§7) must pass —
   `scripts/check-capi-abi.sh`, run by the CI gate.

*This contract is itself versioned by git history. Material changes
to the policy (not just the per-symbol tables) are MAJOR-class
